New in 3.8.
You can no longer send a private message to yourself; trying to now tells you so, the same as you already can't report or demote yourself.
Spamming the global chat now mutes you there automatically: send too many messages too quickly and the server mutes your global chat for a while, timing out on its own like any other mute. Staff are never affected by this. Server hosts can tune how many messages count as spam, how fast they have to come, and how long the mute lasts, or switch the whole thing off, in the server config.
When you start a PVE or PVP game, you now pick a wall reinforce limit right after the map size: 200, 400, or 800 percent of a wall's full strength, or no limit, setting how big a buffer you can build up, with no limit matching how the game has worked lately. In PVE you then choose whether a destroyed wall's bots pile onto the walls still standing, the tougher old swarm, or leave the round as they do now.
Developers can now see every banned account at once with the new /bansall command, a read-only list showing each ban's length and reason. Since bans live on the account and there was no way to list them all before, this makes it possible to review or clear out old bans without knowing the names in advance.
Developers can now silence the whole server at once with the new /muteall command, muting everyone except other developers in a chosen channel, timed or indefinite, and lift it again with /unmuteall. It works like a normal mute applied to everyone, so it saves to accounts and times out on its own, and it asks you to confirm before muting the server.
Developers can now clear the whole server at once with the new /kickall command, disconnecting everyone except other developers, with an optional reason shown to each player as they are dropped. Everyone kicked can sign straight back in, so it empties the server without banning anyone.
Developers can now freeze the whole server with the new /gamestop command and lift it again with /gamestart. A freeze is a complete world-stop: every game pauses where it stands and everyone, other staff and developers included, is locked out of moving, chatting, and everything else until it is lifted, though you can still read back your message buffers while you wait. Everyone hears it happen and is told who froze or unfroze the game.
Restarting or shutting down the server, with /restart and /fastrestart, is now limited to developers, the people who actually host it, rather than every administrator. The staff help now lists these under their own developer-only section.
New in 3.7.
Kicking a player, with /kick, can now carry a reason: type it after the name, for example /kick bob spamming, and the kicked player is told why as they are dropped while the other staff get a quiet note, though it never appears in the public kicked announcement. The kick button on a player's card now asks whether to add a reason, the same way the ban button does.
Any reason you type is now checked before others see it, so a screen reader crash string can no longer be slipped into an AFK, ban, warning, or report reason. If your reason is caught, the action still goes through without it and you are told why, except for a report, whose reason is the whole point, so a caught report is not sent at all.
You can now add a reason when you go AFK: type it after /afk, for example /afk making lunch, and it is announced once when you go away, or leave it off to go away without one.
Staff can no longer put another player AFK; going away is now always your own choice, so no one gets dropped into AFK mid-game without warning.
You can now send a private message to someone who is offline: /pm works just as before, but if the player you message isn't on, your message is saved and delivered to them the next time they sign in, arriving as an offline PM with the usual sound. There's no limit on how many you can leave, and replying with /reply reaches them the same way if they've since gone offline.
You can now report a player to the staff with the /report command, followed by their name and a reason, for example /report bob griefing our walls; it works on someone who has already left too. Any staff online are alerted the moment it comes in, and staff can read back every report, newest first, with the new /reports command, each one stamped with where you were when you sent it.
While a game is paused by the host, the other players can now still use the chat keys and the buffer review keys while they wait, rather than having every key do nothing; movement, the voice key, and the rest stay frozen as before.
Banning a player is now split into two commands so a reason can be added: /ban followed by a name is a permanent ban, /tempban followed by a name and a number of seconds is a timed one, and anything you type after that is an optional reason the banned player is shown and that other staff can see with /bans. The ban button on a player's card walks you through the same choices.
The /restart countdown can now be any length of at least one second, with the old thirty-second minimum and one-hour maximum both removed.
Pressing F4 now also tells you the most players ever connected to the server at once and when that record was set, shown alongside the uptime.
New in 3.6.
When a wall is destroyed, its bots now leave the round instead of piling onto the walls still standing, so the last wall is no longer swarmed by every bot and you can push to much higher rounds.
A new Big map size, 75 by 75, is now offered when starting a game in any mode, sitting between Medium and Large.
There is no longer a limit on how far you can reinforce a wall, so you can build as big a buffer as you can gather the wood for, instead of stopping at four times its full strength.
The tones that play when a player starts and stops talking on voice chat are now heard by everyone nearby in 3D from that player's position, not only by the person talking.
Your email is now private on the information card, shown only to staff instead of to everyone who opens a card.
Preferences now has a Voice chat mode setting on the General tab, where you choose whether your voice streams live or sends as a recorded voice message.
Voice chat can now send your voice as a recorded voice message instead of only streaming it live. Others hear the whole message once you finish speaking, rather than moment by moment as you talk. It still plays in 3D from where you sent it for everyone sharing your space, just like live voice.
Voice chat now picks up a game's audio effects in every mode, not just player versus player. A speaker inside the cave, and your own voice, now reverberate to match the space, while the open-air modes stay dry as before.
The version-mismatch message when you can't connect now tells you which side is behind, so a server running an older version than your game asks for the server to be updated rather than wrongly telling you to update.
For server hosts, the rules agreement can no longer be switched off through the rules version, which now holds at a minimum of 1. A host who genuinely wants players to skip it must set the new ruleswitch option in the server config to 0, so the agreement can never be turned off by accident.
New in 3.5.
The server rules have been brought up to date for everything added recently: the player rules now cover honest team play and looking after your own account, with a reminder that deleting your account is permanent, while the staff rules spell out that renaming or deleting an account, or resetting a password, are heavy last-resort tools and never everyday punishment. The rules version has also moved to 3 to line up with the game, so you are asked to read and agree to them once more.
The /who, /getmotd, and /uptime commands have been removed, since the same information is a single key press away: F1 reads out who is on the server, F6 opens it as a menu, F2 reads the message of the day, and F4 reports the server uptime.
Changing the server's message of the day, with /motd, is now a moderator command instead of an administrator one, since it only edits the message anyone can read with F2 and everyone is still told when it changes. Taking the server down with /restart and /fastrestart stays administrator and up.
Kicking a player, with /kick, is now a moderator command instead of an administrator one, giving moderators a real enforcement action, but it now follows the rank rules the other actions do: you can kick anyone at or below your own rank, players and staff alike, and even yourself, but no longer someone above you. It is still only temporary, so the kicked player can sign right back in.
Muting and unmuting a player, with /mute and /unmute, now takes an administrator rather than a moderator, to match /warn and /ban. Moderators can still look up who is muted with /mutes, but only administrators and up can actually mute or unmute. So across warnings, mutes, and bans, a moderator can view all three while an administrator carries out the action.
A player's information card now shows more details to everyone: their player versus player win and loss record, their best round reached in both the cave defender and environment versus player modes, and their warning count, which used to be visible only to staff. Since the card lists you too when you open it on yourself from the players menu, this is also how you check your own record.
Staff now have two new read-only lookup commands that work just like /warnings: /mutes followed by a player's name shows which channels that player is muted in and how long is left on any timed mute, and /bans followed by a name shows whether they are banned permanently, for a set time with the time remaining, or not at all. Both accept an online player by username or nickname and an offline one by username, and both are moderator and up to view, just like /warnings.
A new account now starts with only the eight files that actually hold a value, its username, email, creation date, nickname, language, pronoun, password, and rank; the room, mute, ban, staff-flag, rules, ignore-list, and warning files are no longer created blank and instead appear on the server only once there is something to store in them. This changes nothing in the game and just keeps each player's folder tidier for hosts who read or edit those files by hand.
Server hosts can now let the owner sign in under more than one name: the config file's owners setting takes a comma-separated list of up to three account names, all treated as the single owner and each forced to developer, matched without regard to case. This replaces the old single owner setting, which is still read for backward compatibility.

New in 3.4.
You can now delete your account with the /delchar command: it warns you first, then asks for your password to confirm, and on success removes the account for good, tells everyone you deleted it, and returns you to the connection menu with that account forgotten on this computer. Administrators can delete another player's account with /delchar followed by their name after a warning, though no one but the owner can delete the owner's account.
Administrators and up can now rename a player's account with the /changename command, for example /changename bob bobby: the new name follows the usual account rules and cannot already be taken. The rename carries the whole account with it, the password, rank, warnings, saved rooms, and the rest, resets the nickname to the new name, and keeps everyone's personal mutes of them pointed at it. You can rename anyone at or below your rank, and the renamed player is quietly returned to the connection menu to sign back in with the same password.
You can now change your password with the /changepass command: it opens a form with three masked fields, your current password and the new one twice, and never shows what you type. On success it returns you to the connection menu and signs you straight back in with the new password. Administrators can reset another player's with /changepass and their name, entering a new password twice without the old one, after which that player signs in with the new password that staff tell them.
You can now change your email with the /changemail command, for example /changemail you@example.com, and administrators can change another player's with /changemail followed by their name and the new email. Every account needs an email to sign in, but you generally sign in with your username and password, not the email itself, and changing it does not sign you out; it shows on the information card.
A player you have personally muted can no longer look up where you are: running /where on you tells them your location is private, and the Location line on your information card reads Private to them. Muting someone limits what they can say to you and see about you, not what they can hear.
Sending a private message to someone who has personally muted you now tells you "This player has muted you," so you know it did not get through, instead of quietly going nowhere. Their information card no longer shows you a private message button at all, since a message would only bounce.
You can no longer personally mute a staff member, from the information card or the ignore menu, and a staff member's chat and voice always reach you no matter what. If you had muted someone before they became staff, that mute simply stops applying to them.
Added two new commands to make managing your personal mutes easier: /ignore opens a menu of the players online to personally mute, and /ignored opens a menu of everyone you are already muting so you can pick one to unmute. These are the same personal mute as the checkbox on a player's information card, silent to the other person, and work from anywhere without opening a menu first.
You can now type /where followed by a player's name to hear where they are on the server, such as in the lobby, a room, or a game, the same location shown on their information card.
When you set a password for a private room, whether creating the room or changing its password later, the game now asks you to type it a second time to confirm and won't set it unless the two match. Joining a private room still asks only once.
New in 3.3.
When you try to chat in a channel you are muted in, the game now names that channel and, for a timed mute, how long is left, such as "You cannot send messages in this global chat channel for three more minutes," instead of the old vague line.
The player information card now shows two more details to everyone: the account's email, listed just under the username, and a member-since date of when the account was created, listed under the pronouns. Accounts made before this was added read their member-since as unknown.
You can now turn the typing sounds off: press F3 while in any text box to toggle the keyboard clicks on or off, and your choice is remembered between sessions.
The preferences menu's Game tab is gone, and its three checkboxes, hearing your own voice while streaming, requiring you to hold the voice key, and asking to exit on escape, now live on the General tab instead.
Your saved account storage and server storage no longer let you add the same thing twice: trying to save an account whose username is already stored, or a server that reuses a saved server's name or address, now tells you it is already there instead of quietly making a duplicate. Editing one onto another's name or address is refused the same way.
Creating an account now asks for an email as well, so the new account form has a username, an email, a password, and a confirm password that must match; the email is saved with your account but is never checked or used to log in. An email is now required, so older accounts made before this update can no longer sign in and must be created again.
Pressing the voice key while your voice is muted now tells you once, right away, that you cannot send voice messages, instead of your held key quietly streaming to the server and being turned back over and over. The microphone never opens and nothing is sent.
Signing back in while muted no longer greets you with the mute alert and sound for each channel you're muted in. The mute is still fully in effect, staff can still see it on your information card, and your messages in that channel are still quietly held back, but you are no longer reminded of it every time you connect.
The mute status line on a player's information card now reads correctly: it lists the team channel alongside global, local, and voice, says "Muted in all channels" when every channel is silenced, and joins the rest naturally, such as "Muted in global, local, and team."

New in 3.2.
The player versus player arena's background ambience is now split to match its reverb: the enclosed cave bed plays inside the walls and the open-air bed outside them, so each side sounds like the place it is rather than sharing one outdoor bed. The readme's player versus player section now also describes the muffled through-the-wall audio between the sides.
In a player versus player game the two sides now hear each other through the walls: footsteps, weapon swings, wood work, and even voice coming from the far side of the wall ring arrive muffled, as if heard through stone, while everything on your own side stays clear. The defenders' interior also carries the cave's natural reverb now, so it rings like the enclosed space it is, while the attackers' exterior outside stays open and dry.
Dropping out of a player versus player match and reconnecting in time now puts you back properly on your side: you return to your team, attackers with their weapon in hand, and any walls that fell while you were away show as the rubble they now are. This is also what lets a reconnection fill an empty side back in, so a paused match resumes instead of forfeiting.
In a player versus player match, if a whole side is left with no players, whether someone left or dropped out, the match now freezes and a forty-five second forfeit countdown begins, spoken aloud in its final seconds. If that side gets a player back in time the match simply carries on; if not, they forfeit and the other side wins the same as any other victory. If both sides empty at once the game just ends with no winner.
The readme now has a full player versus player section, laying out the bullseye arena, the defenders in the center, the ring of four walls, and the attackers in the band outside, and walking through how sides are chosen, how the attack and build windows play out, and how each side wins.
Your player versus player wins and losses are now kept on your account: at the end of each match the winning side each gain a win and the losing side a loss, saved so they carry across sessions, while watchers earn neither.
The player versus player results screen now shows a personal stat for the match as well: attackers see the total damage they dealt to the walls, and defenders see how many pieces of wood they placed to reinforce.
A player versus player match now has a winner: the attackers win by bringing all four walls down, while the defenders win by keeping at least one wall standing until the clock runs out. When the match ends, a results screen names the winning side and shows how many walls fell or held, then asks each player whether to play again. Playing again returns everyone to staging with the sides cleared, so you pick them fresh, and it waits there until enough players are set for even teams.
In a player versus player game, the defending side can now reinforce the ring walls from the inside: during a build window, stand near a wall and press Shift and Enter to spend a piece of wood and mend it, with the placement heard in 3D at that wall just as in the other modes. You can build any still-standing wall back up to the over-build cap, but a wall smashed to nothing is gone for good, so the defenders' whole job is to keep every wall alive. Gathering wood off the ground is allowed at any time, but placing it only during a build window.
In a player versus player game, an attacker's swing during an attack window now truly damages the ring wall it lands on, and a wall battered to nothing collapses into impassable debris, so attackers still cannot cross into the interior through the gap. The four wall strengths also report correctly now on the N, E, S, and W keys as the walls take damage.
New in 3.1.
In a player versus player game the attacking side can now swing at the ring walls: hold Space to swing your weapon. The strike lands as a solid impact during an attack window, or glances off as a ricochet during a build window or the build phase, so you hear which wall you are on and when your swings would count. The walls still take no damage yet, which lets both sides feel the rhythm and reach before the fighting is real.
A player versus player round now runs its timed attack phase after the build phase, swinging between attack and build windows. In an attack window the attackers strike while defenders can only gather; in a build window defenders repair while attackers only glance off, and the clock counts down only while the attackers can act. The walls do not take damage yet, so for now the round simply ends when the clock runs out.
A player versus player round now opens with a build phase. When the host begins, each side is taken to its place, the defenders to the interior with a stock of wood and the attackers out to the exterior with a weapon in hand. You are told which side you are on, and for now this only sets the stage.
Fixed a bug in the player versus player mode where a host stopping the game left the other players stuck inside it, having to leave by hand. Stopping a game now clears it for everyone, in every mode.
In a player versus player game that is waiting to start, a player can now step aside to watch instead, picking the indoor deck above the defenders or the outdoor deck above the attackers. This frees their team slot, so an odd group of three can become an even one versus one with a watcher. You also choose a deck when you watch a player versus player game from the room, since watchers pick where they observe from.
Typing /help on its own as a staff member now opens the player help straight away, the same as it does for everyone else, instead of making you name a page. You still get the staff page with /help staff.
Team chat, on the semicolon key, is now limited to player versus player games, where it reaches only your own side, attackers or defenders. In the other modes everyone in a game is on one team, so team chat only duplicated local chat, and the semicolon key now does nothing there.
In a player versus player game, you now choose your side before the round begins: open the round menu and pick to attack the walls from outside or defend them from inside, and the host can assign anyone's side as well. The host can only begin once the sides are even, a 1v1 or 2v2 with everyone assigned; otherwise the game tells you what is still missing.
A new player versus player mode, PVP, is now in the works: humans fight on both sides of the walls, one team attacking the four walls from outside while the other defends and repairs them from inside, the mirror of the bot modes but with real people on each side. This first piece lets you choose PVP when starting a game and drops you into its new arena, a defender interior at the centre, the four destructible walls ringed around it, and an attacker area outside.
Text fields now play keyboard sounds as you type, with distinct clicks for a normal character, the spacebar, backspace or delete, and enter, plus a separate accent for capital letters, so you hear feedback as you type.
New in 3.0.
If your connection drops while you are in a room, a game, or watching one, the server now holds your place for 45 seconds and keeps trying to reconnect. Get back in time and you land right where you were, with your wood, score, and host duties intact, instead of out in the lobby. If your game or room ends while you are away, or you don't make it back in time, or the server restarts, you come back to the lobby as before.
There is now a team chat channel, opened with the semicolon key while you are in a game, for talking to your own team, with its own buffer and the me action and slash commands like the other channels. For now everyone in a game counts as one team, so it reaches the whole game, but once the player versus player mode arrives it will reach only your side. Staff can mute a player's team chat like any other channel.
In an environment versus player game, the builder bots and the cave's wood now sound muffled and reverberant from outside the walls, as if the inside of the cave leaks out to you through the stone, while your own sounds on the exterior stay clear. It is a first taste of the through-the-wall audio the future player versus player mode will use on both sides.
Placing wood to reinforce a wall now plays its sound in 3D at that wall, so you hear which side you are building on, off to your left when you are at the left wall for example. Everyone else in the game hears each placement at the wall too, so they can tell exactly where the reinforcing is happening.
The inside of the cave now carries a natural reverb, so footsteps, wall hits, wood, and the other sounds down in the cavern reverberate the way they would in a real enclosed cave, with a soft tail of reflections rather than distinct echoes. The open-air outdoor areas and the outdoor watcher deck stay dry, with no reverb.
Your starting wood and your best round bonus wood now arrive as a single random size rather than a mix, so one game you might open with all small pieces and another with all large. The game over reward now names the size you earned, for example thirty two large starter wood, so you know exactly what is waiting for you next round.
In an environment versus player game, the builder bots now follow the same rule when they repair a wall, so a bot that carries a larger piece of wood mends more than one hauling a small piece.
The size of the wood you place now matters when you reinforce a wall: larger pieces add more strength, with small giving five to ten, medium ten to twenty, and large fifteen to thirty, echoing the tiers of the enemies' weapons. Reinforcing always spends your biggest piece first, so seeking out the large wood with the D key is now worth the trip.
A player's information card, opened from the F6 players menu, now shows whether that player has agreed to the server rules, so you can tell at a glance who is able to join a room or game with you.
When you first sign in you must now read the server rules and agree to them before you can play: type /rules to read the player rules, then /rules agree to enter the lobby, or /rules disagree to leave. Everyone must read the player rules to agree, and staff must read both the player and staff rules. You only agree once, and are asked again only if the server's rules are updated.
Staff can now review a player's warning history with the new /warnings command, which opens a read-only list of every warning that player has received, each showing its level, who gave it, and the reason.
Wood now drops in three sizes, small, medium, and large, each with its own sounds and chosen at random as it falls. Pressing D to find nearby wood now tells you each piece's size. Pressing I to check your wood now breaks your total down by size, such as fourteen small, five medium, and twenty-two large pieces.
Reviewing your message buffers now stops firmly at the ends: moving past the first or last buffer, or the first or last message within one, no longer does anything, so you can feel the edges. The exception is a buffer or message list with only one entry, which still reads out each time you press, so it is never left silent.
In a player versus environment game, the enemies now add to their numbers every second round instead of every round, so the difficulty climbs more gently and a run lasts longer.
The main menu now has a documentation menu, listed above the connection menu, where you can read the changelog, credits, readme, and todo list without connecting to a server.
You can now read the server's rules in game with the /rules command: type /rules for the player rules, or /rules player or /rules staff to read either page. Anyone can read both, including the rules staff are held to, so you can always see the standards staff must follow.
For server hosts, the help pages have moved into a docks/help subfolder, named player.txt and staff.txt, with a new docks/rules subfolder reserved for later. Players see no change beyond a cleaner help window title.
The readme now includes a full keyboard reference that lists every key the game uses, grouped by purpose.
The game now comes with a readme that explains how everything works, moving around, chatting, the players menu, rooms, and both game modes. The in-game help, opened with /help, is now just the list of chat commands.

The F6 players menu now shows each player's current language channel beside their name, so you can tell who is on which channel at a glance.
Staff now have action buttons on a player's information card, opened from the F6 players menu: kick, ban, promote, demote, notify, and warn, each shown only when your rank allows it on that player. Choosing one walks you through any details it needs, such as a ban length, a new rank, a notification, or a warning level and reason. It then carries out the same action as the matching command, with the same rules.
The player information card, opened by choosing a player in the F6 players menu, now has a Mute this player checkbox and a Private message this player button. Muting someone hides their global chat, local chat, private messages, and voice from you only, and it stays muted across sessions until you uncheck it. The player is never told.
In an environment versus player game, the round timer now counts down only while you can actually attack, so the rest windows where your swings ricochet no longer eat into your time. The L key now works inside a game too, reading out who is in the lobby, the same as it already does from a room or the lobby itself.
The voice key now behaves the same whether or not hold to talk is on. Both ways are limited to one start or stop every half second, so quickly tapping the key can no longer spam the voice on and off sounds.
Voice chat no longer drives your ping up or drops the connection when two or more people talk at once. Voice is now sent in a way that simply skips the occasional lost moment of audio instead of forcing the whole connection to resend it, so talking stays smooth even over a slower link.
You can now press F6 to open a players menu that lists everyone on the server, each shown with their away status, rank, and name. Choosing which saved server or account to use has moved into the Server storage and Account storage menus, where it is now a Select option, instead of being a separate item in the connection menu.
The F2 and F4 keys have swapped: F2 now reads the server's message of the day, and F4 now reports the server's uptime.

In an environment versus player game, a round can no longer end early for lack of progress. The only way to lose is to run out the wave's time.
Fixed a bug in an environment versus player game where right after you chose to play again, your weapon could strike a wall from well out of range until you moved.
Account passwords are now protected with much stronger, modern encryption. Because of this, every existing account must be created again, as older accounts can no longer sign in.

Staff commands that target a player now accept the name typed in any capitalization, so a command works whether or not it matches the account's exact case.
You can no longer set your nickname to another account's username, so no one can borrow another player's identity while they are signed out.
The server now firmly ties each player to the account they signed in with, closing a loophole that could let a modified client appear as someone else.
New in 2.6.
In an EVP game, press B to hear where the builder bots are, by direction and distance, nearest first, the same way the P key locates other players.
You can now play a brand new game mode, EVP, short for environment versus player, the mirror of the original: instead of defending the four walls you attack them from outside with weapons, while builder bots gather the dropped wood to repair them. Each round opens with a build phase, then a timed wave to bring all four walls down, and every round you clear sends more bots to defend the next. The game ends when the bots hold you off, and you are scored by the rounds you cleared.
Enemy weapon damage is random again instead of a fixed amount, and the ranges are wider than before: the axe does one to three, the bat two to four, the crowbar three to six, and the hammer four to eight, so heavier weapons can land a real blow against a full strength wall instead of barely chipping it.
Starting a game now begins by choosing a game mode, currently just PVE, before you pick who can play and the map size, making room for more modes to come.
Kicking a player from a game now returns them to wherever they came from: a player who joined through your room goes back to the room as before, but one you invited in from the lobby is sent back to the lobby instead.
When you invite a player into your game, you now first choose whether to invite someone from your room or straight from the lobby. This lets you pull a lobby player right into the cavern, skipping the room and its password. A player you invited from the lobby is sent back to the lobby when they leave the game or it ends, rather than being left in your room.
Local chat now reaches everyone sharing your space, whatever language channel they are on, so people in the same room or the lobby no longer have to switch to the same channel just to talk. Global chat is still split by language channel.
You can now set the pronouns used when messages refer to you, with the new slash pronoun command: masculine, feminine, singular they, it, or none, which simply uses your name instead. Creating an account no longer asks for a gender, and your pronouns default to singular they, saved and changeable any time.
Pressing R while in a game now reads out who is waiting in the game's room, rather than only who is in the game with you, so you can see at a glance who you could invite in.
You can now invite and kick players in your games, not just your rooms. If you host an open game, the round menu gains an invite to game option that pulls someone from your room in, answered with /accept or /reject, and a kick from game option that sends a player back to the room. The room menu's own options are now named invite to room and kick from room, so it is always clear which space you are acting on.
New in 2.5.
You can no longer watch your own game. A game you host or are playing in no longer shows up in the watch list, since there would be nothing for you to see.
You can now watch a game instead of playing it. Choose watch game from the room menu to pick any game in the room, including single-player games and ones already underway, and up to four people can watch at once. You float on a deck above the cavern and take no part, but you hear everything and can still chat and use voice, and press escape to stop watching.
You can now invite people to your room. Choose invite a player from the room menu to see who is in the lobby and pick someone; they are told privately and join with /accept or refuse with /reject, within two minutes. Accepting takes them straight in, even past a private room's password, and you are told whether they accepted, declined, or let it expire.
The game over screen now lays out your results in full: the wood you gathered, the rounds you completed, and how long you survived. Beating your best round ever earns a bonus of twenty to forty wood, waiting in your stockpile when the next round begins. To make room for it, the ordinary starting wood has been lowered from fifteen to thirty down to ten to twenty.
You can now adjust the ambience and music volumes while you play: page up and page down change the music, and home and end change the ambience, with each setting remembered for next time.
Each piece of wood you spend reinforcing a wall now adds ten to twenty strength instead of five to ten, so repairs go further.
The limit on attackers has been removed, so every round now adds one more bot to each wall with no upper bound instead of leveling off.
Enemy damage is now a fixed amount per weapon instead of a random range, with the axe doing one, the bat two, the crowbar three, and the hammer four, so a hit always takes off a known, predictable amount.
Pressing tab during a rest now tells you which round you just finished, for example round three complete.

New in 2.4.
You can now choose which recording device voice chat uses. In preferences, under the sound tab, the new select recording device option lists your microphones. Pick one or leave it on Default, and your choice is saved for next time.
The voice chat system has been completely recoded. Your voice now streams live while you hold the talk key, so others hear you in real time as you speak instead of waiting for a recorded clip, and it still plays from your spot on the map. The option to hear your own voice now applies to this live stream, is renamed to hearing your voice when streaming, and is off by default.
The enemies now hit a little softer, especially the heavier weapons. Each weapon's damage has been lowered and made more consistent, with the crowbar and hammer no longer landing the occasional huge blow. So a cluster of bots can no longer strip a well-built wall as quickly, and surviving comes down to steady upkeep rather than bad luck.
You can now save your servers as presets instead of retyping an address every time: in preferences, under the general tab, add a server with a name, an address, and a port, or edit and remove your saved ones, and your list is kept for next time. The connection menu lets you load a saved server, showing which one is currently set, and you can also change the address or port directly without saving a preset. The load list opens on a Custom entry that keeps whatever address you typed in, so a non-preset server now reads as Custom rather than none.
You now get a heads-up before every wave. About ten seconds before the enemies attack, whether it is the first wave or a later one, everyone in the game hears a warning sound and an alert telling you how long until they come, so a wave never catches you off guard.
You can now toggle auto-running with Alt and R wherever you can move: when it is on you run by default and hold to walk, and when it is off it is the other way around, so you no longer have to hold a key to keep up a run. The game says auto running enabled or disabled as you toggle it, and your choice is remembered for next time.
Each enemy wave is now announced with its round number, for example Round 1, the enemies are coming. The count climbs with every fresh attack, so you can tell how deep into a round you are, and pressing tab for the round status reads the number too.
In a game, the host now opens the round menu with enter instead of tab, and anyone can press tab to hear the current round status, such as how long is left in the build phase or whether the enemies are attacking or resting.
The end-of-round message now reads your survival time in minutes and seconds instead of a raw seconds count, for example one minute and thirty-two seconds rather than ninety-two seconds.
You can no longer reinforce a wall while the enemies are actively attacking. Reinforcing now works only during the build phase and the rest periods between attacks, so shoring up a wall is a between-wave decision rather than something you can do in the middle of an assault.
New in 2.3.
The enemies now keep growing their numbers far longer before they level off. Each attack burst still adds one more bot to every wall, but the ceiling has been raised from four per wall to eight, so a long round can build from sixteen attackers up to as many as thirty-two. Simply outlasting the wave is no longer a safe stalemate.
Walls can no longer be reinforced without limit. A wall now tops out at four times its full strength, so you can still stockpile a buffer but can't build one so high the enemies could never break it. Once a wall is maxed, trying to add more keeps the wood in your pack and tells you the wall can't take any more.
An open game can no longer have its round begun while you are the only one in it. Because open games are meant to be shared, the host now needs at least one other player to have joined before starting; to play by yourself, start a single-player game instead.
You can now choose a language chat channel with /channel. Global and local chat are split by channel, so you talk only with others who share yours, while private messages and staff chat reach everyone regardless. Everyone starts on English and their choice is saved, and there is also an Unfiltered channel, advised for adult topics, where the word filter never applies.
During a wave you are now told every time the enemies pull back to rest and every time they surge again, each with its own sound and an alert. This tells you when it is safe to rebuild and when the next push is starting, not only at the very beginning of the wave.
Enemy bots now attack in a measured, capped pattern instead of piling up without limit: each wave opens with one bot per wall, and every attack burst adds one more to each wall, up to four per wall, sixteen in all, and never beyond that. When a wall falls its bots move onto the walls still standing, but still only up to that four-per-wall cap, so no single wall ever gets swarmed.
Strikes on a wall now swell with danger: the weapon blow and the wall's creak start quiet while a wall is strong and grow louder the more battered it gets, so you can tell by volume which wall is in real trouble.
Gathering wood now reaches five tiles: simply walking near wood scoops up anything within five tiles of you automatically, instead of only the tile you land on, so there is no longer a separate grab key to press.
Reinforcing a wall no longer means leaning right against it: standing within five tiles now counts from any direction and reinforces whichever wall you are nearest, no matter which way you are facing. You can also hold Shift and Enter to keep placing wood instead of pressing it for every piece.
Larger maps now grow much more wood. Bigger caverns drop several pieces at a time and hold more on the ground at once, so a team spread across a big map is no longer starved for wood the way a small map never was. Small maps are unchanged.
New in 2.2.
Cavern walls now start each round between half and full strength, instead of as low as a fifth, so no wall begins so weak that the bots can topple it before you have a chance to reinforce it.
Each player now starts every round carrying a random fifteen to thirty wood, a head start so you can shore up the weakest walls right away instead of waiting for the gathering to pick up.
Voice chat now follows people as they move: a clip plays from wherever the speaker is and tracks them around the map while it plays, so you can hear them walk past you. You can move around their voice too, just like footsteps.
A wall that has fallen now reads as destroyed instead of zero percent, both when you check a single wall with N, E, S, or W and in the full summary on T.
If you run your own server, new account folders now keep the exact capitalization the player signed up with, for example EllaShey instead of ellashey, so the folders are easier to recognize. Signing in still works no matter how you type your name.
Commands that target a player now accept either their username or their current nickname, so you can act on someone by the name you actually see. They also require the target to be online, the one exception being unban, which still reaches offline accounts by username. When a name isn't found, the game now tells you whether that player does not exist or is simply not online.
You can now set a nickname with /nickname or /nick, an alternative name shown in place of your username everywhere, for example /nick bob. Type /nick default to go back to your account name. Staff can set another player's nickname the same way, everyone is alerted when a nickname changes, and the who and staff lists show the real username in parentheses.
The player events buffer now collects what others are doing across the server: someone creating a room, joining one, deleting one, leaving one, starting an open game you can join, stopping or leaving a game, kicking a player, or going AFK and coming back. Your own actions still confirm to you in the misc buffer instead, so nothing is doubled up.
You can now press D in a game to hear where the wood around you is, listed nearest first as a direction relative to the way you are facing and a rough distance, the same way P locates players. The four-wall strength summary has moved from D to T.
The looping wood sounds now fall silent while a menu is open in a cavern, such as the round menu or the play again prompt, and resume when you close it.

While you are AFK everyone is told, your name reads as AFK in the lists wherever you are, and administrators and up can put another player AFK with /afk followed by a name.
The owner account named in the server config is now always a developer and can no longer be demoted, so a self-demote can't strip your powers or revert confusingly on your next sign in.

Cavern walls are now sturdier, taking about twice as much punishment before they fall, so a round lasts longer and reinforcing goes further.
Enemy waves now come in bursts: the bots attack for a stretch, then pull back to rest for an equal stretch before attacking again, giving you time to rebuild your walls between pushes. Each attack and rest lasts half of that map's build phase, and the bots keep accumulating, so every burst hits harder than the last.

When a round ends, the game now tells you how long you survived and asks whether to play again or leave. Choose play again to stay in the cavern for another round, or leave to head back to your room. In an open game the next round starts once everyone has answered, and the host begins it.

Anyone can still leave during a pause, and if the host leaves, another player takes over so the game keeps going.

You hear every strike at the wall it hits, growing louder as that wall weakens, and a wall that runs out of strength breaks into rubble you can walk across. The round is lost when all four walls have fallen, and you are told how long your team survived before the cavern resets for another round.
In a game you can now gather wood and reinforce your walls, which each start at a random strength. Wood drops around the cavern as the round runs; collect it by walking over it or pressing Enter, and reinforce the nearest wall with Shift plus Enter. Press I for your wood, N, E, S, or W for one wall, or D for all four, and the game menu has moved to Tab to leave Enter free for gathering.

New in 1.2.
Added a preferences menu, reachable from the main menu, where you can change game, sound, menu, and speech options, choose your sound device, and reset everything to defaults.
Joining the server now requires an account instead of just a name; you can create one from the new connection menu by entering a username and password and choosing your gender. Your password is stored securely and is never saved as plain text.
Added a connection menu to the main menu, where you can sign in or create an account. The connection menu gives you three ways to get on: sign in with the account you last used on this computer, sign in as a different account by entering its username and password, or create a new account.
Pressing F3 to ping the server now plays a sound when the request is sent and another when the reply arrives, so the gap between them lets you hear the round-trip time. The exact ping is still announced in milliseconds.
The sounds folder has been reorganized into categorized subfolders for a cleaner, more organized layout.
Fixed server notifications, such as players coming online or going offline and chat alerts, being announced two or three times instead of once. The chat word filter also works again, so messages containing forbidden words are properly blocked.
Brought back the ability to save your preferences.
Brought back the ability to voice chat with other players.
You can now clear a buffer's contents, which keeps old messages from being included the next time you export that buffer to a log file.
Introduced a more advanced and flexible buffer system to handle various message types in the chat client. The system includes multiple buffer categories such as alerts, chats, and pm's, which allow messages to be sorted based on their type.
New in 1.1.
Made further performance improvements to the server.
Removed the ability to perform web related actions, such as viewing comments, posting comments, and watching server related events as they come in.
Removed the ability to save your preferences.
Improved network performance so the server handles more players and incoming messages more reliably.
Updated the message of the day system for clearer delivery, and admins can now change it with a simplified /motd command that is saved so it persists for everyone.
Made some adjustments to ensure that the server uptime is properly announced based on the time passed since startup.
The server uptime is now more precise and can track multiple time units.
Fixed crashes and unexpected behavior that could occur while sending and receiving messages.
Improved tracking of when players log in and disconnect, so everyone gets more responsive notifications.
The server can no longer play sounds directly on your client for notifications or events; those sounds and commands have been removed.
New in 1.0.
Removed the voice chat feature entirely due to issues that could not be fixed; voice recording, playback, and transmission to the server are all gone.
Added a basic filtering system to the server for banning certain words or phrases written in chat.
Improved how messages are formatted and made sending and receiving them more reliable.
Made some adjustments for showing the server uptime dynamically, reflecting the accurate time intervals in days, hours, minutes, and seconds.
You can now press the F3 key to send a ping request message to the server to measure the average round-trip time.
You can now measure the server's latency, or round-trip time.
You can now press the F2 key to request the server's uptime. The server's uptime will be displayed in a human-readable format such as days, hours, minutes, and seconds.
The server now keeps track of how long it has been running.
You can now press the F4 key to request the MOTD from the server.
Admins can now change the message of the day with the /motd command.
The message of the day is now sent to players whenever they request it.
Improved messaging so that both public chat and private messages are delivered reliably.
Improved management of the player list, logging in, and players going online and offline.
You can no longer have names with spaces!
Voice clips will no longer play simultaneously. Once a voice/sound clip is received, the old one stops, and a new one plays.
Sound uploads to the server are now capped at 30 seconds.
Added sound notifications for chat related events.
Added a message of the day, or MOTD for short, set on the server.
Added a basic menu navigation system and the ability to send messages in the chat.
Added basic networked chat, so players can send messages and read each other's chats.
