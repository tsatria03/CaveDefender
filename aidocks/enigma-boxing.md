The compiled exes are optionally **boxed** with **Enigma Virtual Box** (a free file-virtualization packer; the console tool is `enigmavbconsole.exe`). Boxing embeds chosen DLLs/assets *into* the exe, served from memory at runtime, so the shipped folder has fewer loose files. It is a **build-time step only** — the boxed exe is self-contained and runs on machines without Enigma installed; it is **not** a player runtime dependency. Wired into `build/tools.py` as a post-compile stage.

**What is embedded vs. kept as a real file** — a deliberate hybrid:
- **Client `cfc.exe`** embeds the 5 audio DLLs (`bass`, `bassmix`, `bass_fx`, `opus`, `phonon`) **plus the entire `sounds/` and `docks/` folders**. Kept real in `lib/`: the 4 screen-reader DLLs (`Tolk`, `nvdaControllerClient64`, `SAAPI64`, `zdsrapi`) and the third-party `GameEngine64.dll`.
- **Server `cfs.exe`** embeds 4 audio DLLs (`bass`, `bassmix`, `bass_fx`, `phonon` — **no opus** (voice is client-only), no `GameEngine64`, no asset folders). Screen-reader DLLs stay real.
- **Why the split:** audio DLLs load by bare name and are AV-boring → safe to embed. **Screen-reader DLLs stay real** — a silently-unresolved one means *no speech at all* (catastrophic for this audience) and Tolk loads them via paths that expect real files. **`GameEngine64.dll` stays real** — AV-sensitive (a PureBasic DLL doing HTTP/FTP/email is a heuristic flag inside a wrapped exe), loads CWD-relative, and is third-party (easier to hot-swap loose). Assets embed fine: sounds load in-process, and `dockread`'s `file_exists()` gate is satisfied by Enigma's virtual filesystem.
- **Result:** client ships as `cfc.exe` + a `lib/` of 5 real files (4 screen readers + GameEngine64), **no loose `sounds/`/`docks/`**; server as `cfs.exe` + a `lib/` of 4 screen readers.

**The `.evb` project files** live in **`src/client/cfc.evb`** and **`src/server/cfs.evb`**. An `.evb`'s *location* is independent of the source paths inside it — they sit in `src/` but reference the assets in `cf/<side>/{lib,sounds,docks}`. Enigma has **no wildcard support**, so every embedded file needs its own explicit `<File>` entry (the client `.evb` is ~140 KB / 157 entries: all 148 sounds + 4 docks + 5 DLLs).

**Regenerating the client `.evb`:** because it enumerates every sound/doc, it is **generated, not hand-edited**, by **`scripts/gen_cfc_evb.py`** (`python scripts/gen_cfc_evb.py` — renamed from the earlier gencfcevb.py; a `scripts/gen_cfs_evb.py` now exists for the server side too). It reads `build/tools.ini` for name/password and WALKS `cf/client/{lib,sounds,docks}` generically (no hardcoded subfolders, so folder reorgs like 2026-07-16's `equipment/` restructure need only a re-run, never a script edit), writing `src/client/cfc.evb`. **RULE: re-run it whenever client sounds or docks are added/removed/renamed** — the `.evb` is a snapshot, so a new sound not regenerated in won't be embedded (it would ship loose, or go missing after the strip). Last regen 2026-07-16: 241 sounds + 4 docks + 5 DLLs, reflecting the equipment/ tree and the 12-weapon roster. Relates to [[sound-placeholders]]: when a new sound is wired up, the `.evb` needs regenerating before it ships boxed.

**`build/tools.py` integration:** boxing is a first-class pipeline stage alongside compile/package/release — menu option **"Box only"**, and full-release prompts *"box the project?"*. Toggle with **`box_project`** under `[game]` in `build/tools.ini` (defaults on; the legacy `box_client` key is still honored). The `enigmavbconsole.exe` path comes from `~/.game_tools/tools.ini` `[tools] enigma=` (falling back to the default install path). Per side it runs `enigmavbconsole <evb> -input <exe> -output <boxed>` — the `-input/-output` overrides mean it never depends on the `.evb`'s hard-coded release paths and never reads+writes the same file — then **swaps `<out>_boxed.exe` in under the real name**, **strips the embedded DLLs from the build's `lib/`**, and (client only) **deletes the embedded `sounds/`/`docks/` folders from the build**. The strip touches only the release output; the `cf/<side>/` staging it reads from is untouched, so every cycle is clean. (The non-interactive CLI now takes **5** flags: `<skip_compile> <skip_box> <skip_package> <skip_release> <skip_empty_release>`.)

**Caveats:** the boxed exe passed Windows Defender locally, but SmartScreen reputation is per-file-hash — a fresh boxed exe may prompt "not commonly downloaded" on another machine (reputation, not detection). Embedded assets and `GameEngine64.dll` resolve **CWD-relative**, so a normal launch (cwd = exe folder) is required. Enigma virtualization is **packaging, not protection** — public unpackers can extract embedded files; it is not a substitute for NVGT encrypted packs (not needed here — most sounds are shared across the dev's other games, so extraction is a non-issue).
