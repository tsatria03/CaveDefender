There is **no test suite and no linter** — running is scripted by the Python launchers in `cf/<side>/`, and compiling/packaging/releasing by `build/tools.py`, but it's otherwise manual, on purpose. The launchers hard-code `C:\nvgt2\nvgt.exe`; `build/tools.py` reads the engine path from `~/.game_tools/tools.ini` (pointed at the same nvgt2 build — see [[engine-pinned-to-nvgt2]]). All need Python installed, and the console ones expect `.py` associated with `python.exe`. Each launcher runs the `src/<side>` script with cwd set to its own `cf/<side>` folder so assets resolve (see [[path-conventions]]).

- **Client:** `cf/client/cfc.py` runs `src/client/cfc.nvgt` (cwd `cf/client`). It launches `nvgt.exe` with `CREATE_NO_WINDOW` (no console) and its stdout/stderr captured to a temp log, then **watches for `COMPILE_WAIT` seconds (default 5)**: a failed compile exits fast with a non-zero code, so the launcher writes the captured output to **`errors.txt`** in the directory cfc.py was launched from (plus a popup) — since the windowless client otherwise swallows compile errors; a clean compile keeps running, so the launcher detaches and leaves it (deleting the temp log / any stale `errors.txt`). The launcher hides its own console at startup (ctypes `ShowWindow`), so the `COMPILE_WAIT` watch runs invisibly in the background — the client's own NVGT game window opens right away with **no lingering cmd window**, and an absent/empty `errors.txt` means "no compile errors". (Server differs — its launcher keeps the console open, so it shows compile errors live and needs none of this.) `main()` installs the keyhook, gates on `SCREEN_READER_AVAILABLE` / `SOUND_AVAILABLE`, blocks a second instance via `gamstence.is_already_running`, loads the game-engine DLL (`load_engine()`), creates the message buffers, reads prefs, sets up the network client, then loops `mainmenu()`.
- **Server:** `cf/server/cfs.py` runs `src/server/cfs.nvgt` (cwd `cf/server`) with blocking `subprocess.run`, so its cmd window stays open the whole session (it's the server's UI). The server loads config from `data/preffs/conf.svr` and the MOTD from `data/preffs/motd.svr`, ensures the `data/` folders exist, calls `net.setup_server()`, sets a `show_game_window` title (version/port/user-limit) then hides that NVGT window with `if(!SCRIPT_COMPILED) hide_window()` (interpreted = headless, the cmd window is the UI; a compiled build keeps the window), then loops `netloop()` + `gametick()`.
- **Build a release:** `build/tools.py` (launched by `build/tools.bat`, or run with flag args non-interactively — the CLI is now **5** flags, see [[enigma-boxing]]) is the unified pipeline that replaced the old per-side `cfcm.py`/`cfsm.py`. Its **compile** step does both sides in turn: it first mirrors `build/version.txt` into both `src/<side>/includes/version.nvgt`, then runs `nvgt -c` from each `src/<side>` (bundle lands there), copies that side's assets from `cf/<side>/` into the bundle (client: `lib,sounds,docks`; server: `data,docks`), and moves it to `release/windows/CaveDefender<Side>_password_is_<pw>/<cfc|cfs>`. **Server security scrub:** after the server bundle is assembled, `sanitize_server_bundle(SERVER_BUILD)` strips private/host-specific data from the shipped bundle so the itch.io archive is a clean slate: (1) empties `data/players` (removes every account — passwords/emails/records — but keeps the empty folder), (2) blanks the `owners=` line in `conf.svr` (so no username ships pre-trusted as owner — else whoever registered that name on the new server would become owner; the host sets their own), (3) deletes `reports.svr` + `feedback.svr` (private player-submitted logs — reports name players, feedback carries names + messages) and `peak.svr` (this server's peak-players stat) if present, and (4) overwrites `motd.svr` with the neutral default `Welcome to CaveDefender. We hope you enjoy your stay here.` (no trailing newline — the whole file is the MOTD). Warnings need no handling: they live per-account in `data/players/<name>/warnings.usr`, already gone with (1). Bundle copy ONLY — the source under `cf/server` is never touched (verified: source `conf.svr` keeps `owners=tsatria03,tony`, source `motd.svr`/`peak.svr` unchanged). The host's server recreates/repopulates all of this on its own. Note `conf.svr` uses LF endings; the scrub preserves each line's ending. Full `preffs/main` inventory: KEEP conf/chatfilter/languages/pronouns (config); SCRUB reports/feedback/peak; RESET motd to the default welcome. **Package** makes two password-protected `.7z` archives (client + server) in `release/archives/`; **release** attaches both to a GitHub release. (SimpleFighter's website-update step was removed — CaveDefender isn't published on the site.) The engine/7-Zip/gh paths come from `~/.game_tools/tools.ini` (shared across the legacy-engine games), the per-repo config from `build/tools.ini`. **Boxing** (Enigma Virtual Box) is a post-compile stage — see [[enigma-boxing]].

Remember [[dont-compile-yourself]]: never run the launchers/compilers; the dev builds and verifies.
