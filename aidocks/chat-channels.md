Chat is four channels, each a different audience:
- **Global** — key `/` — reaches the whole server. This is the existing public chat (channel 1, `playerchat.ogg`).
- **Local** — key `\` (backslash) — reaches everyone on your **current map instance**. ONE channel that automatically covers lobby / room office / game: it follows you, so you never choose lobby-vs-room-vs-game chat — it's whoever shares your exact space. (`localchat.ogg`, renamed from mapchat.ogg.)
- **Staff** — key `'` (apostrophe) — reaches all ONLINE staff (rank >= moderator), server-wide. Staff-only to send and receive. (`staffchat.ogg`)
- **Team** — key `;` (semicolon) — reaches everyone on your **side within a PVP game** (**PVP-ONLY** as of the PVP build). Attackers hear attackers, defenders hear defenders. In PVE/EVP everyone is one team, so team chat would just duplicate local chat — so there the `;` key **does nothing** (client gated on `game_mode=="pvp"` + `!watching`) and the server rejects `teamchat`/`teamme` outside a PVP game. (`teamchat.ogg`)

**Local chat DONE:** scoped by the server's `room_id` (lobby = room_id 0; each room = its members). Client: backslash key in `game()` -> `comparse(msg, "local")`; `send_chat`/`send_me` route to `localchat`/`localme`. Server: `relay_to_local(room_id, msg)` + `localchat`/`localme` handlers relay `localmsg <display>: <text>` / `localmsg <display> <action>` to everyone with the same room_id; client plays `localchat.ogg` into the **local chats** buffer (created in cfc.nvgt for everyone). Not rank-gated. NOTE: it scopes by room now; when games add their own sub-instances (caverns), local will need to scope by game instance, not just room_id (part of Section 5).

**Every chat input supports slash commands (requirement):** opening any chat (global, local, staff, team) and typing should route through the command parser just like the `/` global chat does — `/me` and all other commands work from every chat input. `/me` and plain text are **scoped to that input's channel** (so `/me waves` in staff chat is a staff-only emote); channel-independent commands (`/who`, `/pm`, `/kick`, ...) behave the same regardless. Implement by giving `comparse()` a channel parameter; only the plain-text and `/me` branches change per channel, the rest stay as-is.

Global chat sound is now `globalchat.ogg` (renamed from playerchat.ogg). Room creation plays `roomcreate.ogg` (sent via the server `play` message on create only).

**Why Local is one channel (resolved confusion):** "map" vs "room" vs "game" chat are the same idea — "talk to everyone here with me." Each space (the lobby, each room's office, each game's cavern) is its own **instance** — a room gets its OWN copy of the office layout, so two rooms using the same layout are still separate scopes and never hear each other. So Local = your current instance covers it all. A separate "room-wide" channel (in-game, also reach the office) was considered and dropped as niche; Global covers reaching beyond your space.

**Buffers:** rename the existing `chats` buffer to **global chats**; add **local chats**, **staff chats**, **team chats**. The global chat input dialog title becomes "Global chat".

**Staff chat DONE:** `comparse()` now takes a `channel` param ("global"/"staff"); `send_chat`/`send_me` route plain text and `/me` per channel (global -> channel 1; staff -> `staffchat`/`staffme` on channel 0), all other commands unchanged. Apostrophe key in `game()` opens the Staff chat input (gated on `myrank != "player"`). Server: `relay_to_staff()` + `staffchat`/`staffme` handlers (gated `is_staff`) relay `staffmsg <display>: <text>` / `staffmsg <display> <action>` to online staff; client plays `staffchat.ogg` into the **staff chats** buffer. That buffer is created in cfc.nvgt at startup but **removed for non-staff at login** (and on demote; re-added on promote) via `remove_buffer`/`ensure_buffer` (buffer.nvgt) called from the `yourrank`/`rankchange` handlers — so non-staff never have it in their buffer navigation. This `comparse(channel)` plumbing is the foundation local/team chat will reuse. NEXT of the channels: local chat (needs Section 5 scoping), then team.

**Team chat DONE (built before PVP as prep):** semicolon key in `game()` (both the normal and AFK-away key blocks), gated on the client `in_game` flag -> `comparse(msg, "team")`; `send_chat`/`send_me` route to `teamchat`/`teamme` on channel 0. Server: player class gained `string team=""` (routing side; empty in co-op = whole game, PVP sets attack/defend) and `int64 mute_team`; `relay_to_team(sender_index, msg)` sends to same-room_id + same-team players (respecting personal ignore); `teamchat`/`teamme` handlers require being in a game (`get_game_index(room_id)>=0`), check `is_muted_now(index,"team")`, filter like local (crash + word), and relay `teammsg ...`. Client plays `teamchat.ogg` (placeholder — file to be added) into the **team chats** buffer (created in cfc.nvgt). Staff-mutable: `team` added to `valid_mute_channel`, `mute_channels_for("all")`, `get_player_mute`/`set_player_mute`, and the /mute error string; on-disk mute is generic (`mute_team.usr`). NOT rank-gated (all players). Until PVP it behaved like a second local channel (co-op = one team). **Now gated PVP-ONLY** (during the PVP build): the client `;` key is gated on `game_mode=="pvp" && !watching` (new client `game_mode` flag set from the `gamejoined` mode field), and the server `teamchat`/`teamme` handlers require `games[gi].mode=="pvp"` — because in co-op it just duplicated local chat. PVP sets each player's `team` to attack/defend (via the staging side-pick) to narrow it; `set_room_scope` clears `team` on any space change so a side never leaks into another game.

**Build order (by dependency):** 1) **Staff chat** FIRST — server-wide, staff-only, no scoping needed (relay to all online staff); also do the `chats -> global chats` rename here. 2) **Local chat** — comes with the scoping system (Section 5; needs to know who's in your instance). 3) **Team chat** — last (needs teams). Implementation pattern: each channel is a tagged message like `/pm` — client sends e.g. `staffchat <msg>`, server checks permission and relays to the audience, client routes to the matching buffer + sound.
