**BUILD STATUS (as of 3.2): fully built and playable end to end.** Delivered in incremental pieces across 3.1–3.2 (see docks/changelog.txt): arena + mode selection, staging/side-assignment + watcher decks, round flow (build/prep → attack/build window rhythm on the frozen attacking-seconds clock), combat + wall lifecycle (ring walls, impassable `wallwoodpile` debris, defender inside-repair), win/loss + results screen + persistent `wins.usr`/`losses.usr`, desertion→pause→45s forfeit grace + PVP reconnect restore, and cross-wall lowpass muffle + interior reverb (footsteps/swings/wood/voice via a `crosspool` + `pvp_crosswall` position test; wall impacts kept clear). The spec below is now the AS-BUILT reference. Key impl notes: side stored in `player.team` (server) + `my_side` (client, from the `yourside` msg); ring geometry `wmin=size/4, wmax=size-wmin` shared by `spawn_ring_walls`/`draw_map("pvparena")`/`wall_in_dir`/`ring_wall_in_reach[_inside]`; forfeit via `pvp_deserted`+`resolve_pvp_forfeit` polled in `gametick`. The EVP swing-broadcast "gap" the spec mentions was already closed (the fire/windup is relayed).

**Prerequisite status:** the reconnect-persistence foundation is BUILT (see [[reconnect-persistence]]) — the generic 45s hold-and-restore; PVP's desertion pause + forfeit is layered on top of it.

Agreed design for **PVP (Player versus Player)**, CaveDefender's flagship mode (see [[pvp-mode-timing]]). Built across 3.1–3.2. It is the Wallbreaker wall fight ([[game-vision-wallbreaker]]) with **humans on both sides** instead of bots: one team ATTACKS the four walls with weapons (like EVP players), the other DEFENDS by gathering wood and repairing (like PVE players). No bots. It is the clean mirror of [[game-mode-evp]], but with humans on both sides, a REAL physical inside/outside separation, team chat, and muffled cross-wall audio.

**Teams & players.** Max 4. **Strictly EVEN teams: 1v1 (2 players) or 2v2 (4 players)** — no odd/lopsided games (no 2v1 or 1v2; the dev "has a thing against odd ones on the field"). Minimum 2 to start. A 3rd player is a **watcher** (or the group waits for a 4th to make 2v2); an odd member count can't start until one becomes a watcher. **Roles are FIXED for the whole match** (no swapping sides). **Side assignment:** players PICK their own side (attack or defend), and the HOST can override/move anyone before the round starts (self-joiners pick like everyone else). The host **cannot start** until every player is assigned and both sides are filled evenly (message says what's missing).

**Watchers** choose the **indoor (concrete)** or **outdoor (gravel)** overhead deck (not both — hearing both at once is chaotic). They don't count toward teams; they see the whole fight via spatial audio. Local chat reaches everyone in the game including watchers.

**Map geometry (NEW).** PVP needs a map that doesn't exist yet: a single, larger **concentric** arena holding BOTH areas at once — interior floor (concrete, defenders) in the **center**, the four `wallwood` **destructible walls** as the middle ring, an **exterior band** (gravel, attackers) around them, and an **indestructible outer boundary** so attackers can't wander off. Plus the watcher deck over it. This is unlike PVE/EVP, which run on ONE grid; **EVP's "outside" is only a cosmetic reskin** (gravel floor + exterior ambience) of the same interior grid — the EVP player and the builder bots are literally on the same grid, and the bots ignore the player by AI, not by any wall. PVP is the first mode with a genuine inside AND outside separated by the walls. Spaces in this game are isolated instances you teleport between via menu actions (not one connected world), but PVP's interior and exterior are ONE shared space so both sides coexist and the cross-wall audio works.

**Spawns:** defenders at the interior center (like PVE); attackers out in the exterior (exact spread a build detail).

**Geometry numbers (as built, piece 1):** wall ring inset by `size/4` (client `draw_map("pvparena")` + server `spawn_ring_walls` share the formula `wmin=size/4, wmax=size-wmin`). So Small(25): interior coords 7–18 (12×12), Medium(50): 13–37 (25×25), Large(100): 26–74 (49×49); the attacker gravel band is the frame around it (5/11/24 tiles thick per side). It's **concentric (a bullseye), NOT two side-by-side halves** — defender center square, wall ring, attacker outer band; by floor AREA the attacker frame is actually the LARGER region (~2.7× on Medium). A `size/4→size/6` inset would widen the interior if wanted (offered, not taken). **README MUST clearly explain this layout** (inside=defenders, outside=attackers, walls are the front line nobody crosses, demolished wall = impassable debris not a walkable breach) when the PVP docs piece lands — the inside/outside-vs-halves distinction confused even the dev during testing.

**Round structure.** Opens with a **build/prep phase**: defenders gather wood and reinforce the walls; attackers wait outside (can move/position but can't damage yet). Then the **timed attack phase** with an EVP-style **attack/build window rhythm**. The windows are **game-wide and mutually exclusive**: during an ATTACK window attackers damage walls and defenders can ONLY gather (no placing); during a BUILD window defenders place/repair and attackers' swings just RICOCHET (no damage). The two sides **never act on the walls at the same time**. Gathering wood is always allowed.

**Clock.** Uses **EVP's clock**: a budget of actual ATTACKING seconds that FREEZES during build windows, scaled by map size. Single decisive round — **no multi-round/escalation** (nothing to ramp: fixed human headcount, no bots to add).

**Wall lifecycle.** Walls start at a random 50–100% (like other modes; symmetric — both sides face the same config in a match). **Attackers attack from OUTSIDE only; defenders repair from INSIDE only; nobody crosses.** A wall that's damaged but **STANDING (health > 0) can be repaired** (up to the over-build cap); a wall **DEMOLISHED to 0 is permanent** — it can't be rebuilt (matches PVE's "That wall is destroyed") and leaves **IMPASSABLE debris** (attackers can't walk through into the interior — the dev explicitly wants no "weapons blazing" invasion; it'd be unfair). This is a PVP-specific difference from PVE, where downed-wall rubble is a *walkable* breach. Defenders' whole job is **triage: keep every wall above zero.** A single wall is one object with one health bar (no separate inner/outer face).

**Win/loss.** Attackers win by bringing **all four walls down**; defenders win by keeping **at least one wall standing** until the clock expires. No draws. A **forfeit** (see desertion) counts as a normal win/loss.

**Weapons/wood.** Attackers use the weapons (axe/bat/crowbar/hammer, `enemy_damage` tiers). Defenders use the sized-wood reinforce system (small/medium/large scaled repair, largest-first).

**Audio.** (1) All weapon action — **swing (windup), impact, and recovery** — plus movement is broadcast positionally so teammates and the other side hear it; **fix the swing/recovery broadcast gap in EVP too** (today only wall impacts and footsteps carry; a swing that misses a wall is silent to others). (2) **Cross-wall audio carries but is LOWPASS-MUFFLED**, relative to the listener: each side's area is muffled only for listeners on the OTHER side (your own side stays dry). Applies to footsteps, weapon swing/recovery, and voice. Implement via a dedicated muffled sound-pool routed by listener-vs-source side, using NVGT's mixer lowpass FX (the primitive from SimpleFighter's `effect_space.nvgt` — reuse the `pool.mixer.set_fx("$lowpass:...")` idea, SKIP its zone-builder/preset/.sif editor since PVP geometry is a fixed inside-vs-outside check). Drop the cutoff well below SF's ~11000 Hz default for a "through stone" muffle. (3) **Cave-interior reverb** (all three modes): interior concrete = reverb, exterior gravel = dry; inside watcher deck reverb, outside deck dry. **STATUS — reverb AND the source-side lowpass muffle are BUILT for EVP; PVP reuses the same mechanism across the walls.** How it's wired (client `effect_space.nvgt`): the local player's OWN listening pools (`p`, beaconpool, vpool, ambpool, musicpool) are **listener-based** via `apply_all_effects` + a reverb effect_zone spawned only for the inside cavern/deck (`spawn_cave_reverb` in `map.nvgt`), so they're wet inside and dry on the exterior (and never lowpassed — you're right there). The cave-interior SOURCES are **source-based** instead: bot footsteps play on a new `botpool` (net.nvgt `botstep` handler) and all wood take/place plus ground-wood drop/loop play on `objpool` (net.nvgt `footstep` handler routes `objects/items/...` paths there). `set_cave_source_fx(pool, key, reverb_on, lowpass_on)` sets both mixer slots at once: **reverb** on whenever `in_cavern` (source inside), and **lowpass** (`cave_lowpass_fx`, cutoff ~1000 Hz, tunable) on when `in_cavern AND listener_outside` (both flags set in `draw_map`; `listener_outside` = the `outside`/exterior flag). So an EVP attacker on the gravel exterior hears the bots + wood **reverberant and muffled through the wall** (reverb = a tail of reflections, NOT distinct echoes — those are the separate `echo` effect), while their own sounds stay clear — the source-based fix a position zone can't do in EVP (bots and player share one grid; "outside" is a reskin). **PVP:** same pattern, but with real geometry — route each side's sources through pools that carry reverb + `$lowpass` for listeners on the far side of the walls.

**Team chat.** Build the `\` **team channel as part of PVP** ([[chat-channels]] — held off until now because it lacked a real use case): attackers get a private channel, defenders get a private channel (watchers TBD — team chat or just local). Local chat still reaches the whole game (both sides + watchers), which is exactly why each side needs a private channel.

**Results & scoring.** On a win: results screen announcing the **WINNING SIDE**, then per-role stats — **attackers see WALLS DESTROYED (of 4)**, **defenders see WALLS STOOD (of 4)** as the headline, plus damage dealt / wood placed. Then a play-again prompt (per member) → back to staging with **sides cleared for re-picking**; if too few remain to fill both sides, it can't restart and waits. **Persistent per-account record: `wins.usr` and `losses.usr`** (aggregate team win/loss tally, following the folder file-per-field account pattern), incremented at match end; forfeit counts; watchers get nothing. Natural display later: the results screen and the player info card.

**Desertion / disconnect (BUILT).** If a side still has ≥1 player (e.g. 2v2 drops to 2v1), the match **continues uneven**; host migration keeps a host. If a whole **side empties**, the game **PAUSES** (reuses the host-pause freeze) for a **45-second grace window** with a countdown (`pvp_deserted` + `resolve_pvp_forfeit`, polled in `gametick`); if the side is filled again, resume; if the window expires still empty, **FORFEIT** — the remaining side wins → results. Both sides empty → game ends, no winner. This was built on the **reconnect foundation** ([[reconnect-persistence]], the generic 45s hold-and-restore, shipped 3.0) — the old "immediate removal + host migration + back to lobby on disconnect" problem is fixed for ALL modes, which is what makes "come back to your side" possible.
